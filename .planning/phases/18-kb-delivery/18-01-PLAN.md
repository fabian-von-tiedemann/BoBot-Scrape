    1. Create a temporary directory for the KB repo
    2. Initialize git repo
    3. Copy converted/ contents to the repo
    4. Create a README.md explaining the knowledge base structure
    5. Add all files and create initial commit
    6. Add remote and push to GitHub

    Commands:
    ```bash
    # Create and initialize
    cd /tmp && rm -rf bobot-kb && mkdir bobot-kb && cd bobot-kb
    git init

    # Copy converted content (preserving folder structure)
    cp -r /Users/fabianvontiedemann/kod/BoBot-Scrape/converted/* .

    # Remove .DS_Store files
    find . -name ".DS_Store" -delete

    # Create README
    cat > README.md << 'EOF'
    # BoBot Knowledge Base

    Converted routine documents from Botkyrka kommun's intranet (BoTwebb).

    ## Structure

    Documents are organized by category (verksamhet):
    - `Bemanningsenheten/` - Staffing unit procedures
    - `Boendestöd/` - Housing support procedures
    - `Dagverksamhet/` - Day activity procedures
    - `Gruppbostad/` - Group housing procedures
    - `Hemtjänst/` - Home care procedures
    - `Hälso- och sjukvård/` - Health care procedures

    ## Document Format

    All files are Markdown with YAML frontmatter containing:
    - `title` - Document title (Swedish)
    - `source_file` - Original file path
    - `source_url` - Original download URL
    - `category` - Top-level organizational unit
    - `subcategory` - Specific procedure category
    - `updated_date` - Last update date (if available)
    - `document_type` - rutin, blankett, etc.
    - `summary` - AI-generated summary
    - `keywords` - Relevant keywords
    - `topics` - Broader topic categories

    ## Source

    Generated by [BoBot-Scrape](https://github.com/fabian-von-tiedemann/BoBot-Scrape)
    EOF

    # Stage and commit
    git add .
    git commit -m "Initial KB with converted Botkyrka kommun documents"

    # Add remote and push
    git remote add origin https://github.com/fabian-von-tiedemann/bobot-kb.git
    git branch -M main
    git push -u origin main
    ```
